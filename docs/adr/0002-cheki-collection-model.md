# 체키 컬렉션 모델: 캐릭터×이벤트데이 세트축 + 프레임-only 등급

## 결정

게임의 핵심 수집물인 **체키**를 다음 모델로 정의한다.

- **수집 주체(캐릭터)**: 옥자, 시온이를 각각 독립된 수집 캐릭터로 둔다(**옥자는 메인 교감·수집 캐릭터(라이브)**, **시온이는 옥자 곁 컴패니언 겸 수집 캐릭터**). 이후 바나·멜·미호(메인) + 코코·선아·수아·규종이(펫)가 같은 틀로 확장된다.
- **컬렉션 칸(룩)**: `캐릭터 × 이벤트 데이`가 한 칸. 이벤트 데이는 서로 **등급을 매기지 않고(줄세우지 않고)** 동등한 "세트(가로축)"로 둔다. 실물 이벤트마다 의상이 달랐던 자산이 그대로 콘텐츠가 된다.
- **등급(세로축)은 오직 프레임으로만 구분**한다. 캐릭터 아트는 일반·나비가 동일하다.
  - **일반체키** = 표준 프레임(전 카드 공용). 평소 플레이로 자주 획득.
  - **나비체키** = 그 이벤트 데이 테마에 맞춘 프레임(예: 항공 데이 → 공항/비행기). 마일스톤·연속출석 등 정성 보상으로 드물게 획득.

## 이유 / 트레이드오프

- **세트축과 등급축 분리**: 이벤트끼리 레어도를 매기면 특정 이벤트가 천대받는다. 분리하면 "전부 모으기(브레드스)"와 "최애 룩을 나비로 승급(체이스)"의 두 수집 동기를 모두 얻는다.
- **프레임-only 등급(아트 공유)**: 등급마다 새 일러스트를 그리면 작업량이 배수로 폭발한다. 같은 아트에 프레임만 교체하면, 이벤트 1개당 추가 자산은 "캐릭터별 의상 도트 + 이벤트 테마 프레임 1장"뿐이다(표준 프레임·시온이는 영구 재사용). 1주 데모에서 생존 가능한 유일한 구조.
- **등급 2개(일반/나비)로 고정**: 실물 카페에 이 둘만 존재 → 설명 불필요, 직관적. 추가 등급은 작업량 배수라 데모에선 배제.

## 폐기된 대안

- 나비를 *나비 파티클/포일 셰이더/시온이 합성*으로 차별화 → 차별점이 "의미"가 아닌 "VFX"라 약하고, 합성·여백 제약으로 작업이 늘어 폐기.
- 현금 가챠 기반 등급 → 팬게임 정서·카페 이미지에 독이라 배제. 현금은 "현실 방문/굿즈 → 게임 한정 체키 해금"의 오프라인 보상 다리로만 흐른다.

## 향후 확장

- "한정(限定)"은 등급(파워)이 아니라 **출처 표식**으로, 실제 방문·굿즈로만 해금되는 체키에 부여한다.
- 시온이 등 펫의 "스페셜 카드" 추가 여지를 남겨둔다.
